# Week 01 Learning Path

目标：建立游戏设计的基本语言，能用 MDA 分析一个游戏，并能把一个机制转化为玩家动作、关卡结构和开发需求。

## Day 1: 建立框架

- 阅读 MDA。
- 输出：用 MDA 分析一个熟悉的游戏。
- 产物：`03_ai_reading_notes/game_design/paper_mda.md`

## Day 2: 机制词汇

- 浏览 Game Mechanic Explorer。
- 选择 3 个机制：收集、解锁、交换。
- 输出：3 张机制卡。

## Day 3: 关卡基础

- 阅读 The Level Design Book 的 blockout / layout / flow 相关章节。
- 输出：一个小关卡的路径草图说明。

## Day 4: 叙事基础

- 阅读 Emily Short 博客中关于互动叙事或对话系统的一篇文章。
- 输出：一张叙事卡：选择与后果。

## Day 5: 开发基础

- 阅读 Game Programming Patterns 的 Component 或 State 章节。
- 输出：把一个机制写成开发需求。

## Day 6: 案例拆解

- 选择一个公开 GDD、postmortem 或开源小游戏。
- 输出：`05_case_studies/` 下的一份案例拆解。

## Day 7: 项目调用

- 选择一个当前项目。
- 更新该项目的 `context_pack.md`。
- 用 prompt 生成一个小型策划方案，并记录哪些资料被调用。
