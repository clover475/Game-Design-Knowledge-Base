# 这份资料讲什么

这本书解释游戏开发中常见的程序结构和模式，适合策划理解“一个机制最终需要什么状态、事件、对象和系统支持”。

# 初学者重点

- 策划写机制时要明确状态、事件、触发条件和反馈。
- 开发需求不只写“实现某功能”，还要写数据、规则、边界和调试方式。
- State、Component、Observer、Command 等模式常用于角色、交互、输入、事件和 UI。

# 可转化方向

- 玩家动作：输入与命令。
- 游戏机制：状态机、规则系统、组件组合。
- 关卡设计：触发器、区域状态、事件监听。
- 叙事设计：任务状态、对话条件、分支记录。
- 美术需求：动画状态、特效触发、UI 状态。
- 开发需求：模块、数据表、事件流、可调参数。
